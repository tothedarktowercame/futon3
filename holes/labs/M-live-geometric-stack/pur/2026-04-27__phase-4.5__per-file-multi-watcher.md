actions taken (single session, 2026-04-27):
- Created `futon3/scripts/ingest_one_file.clj` (252 LOC). Per-file
  ingest path; vendored Clojure projector + dispatch to elisp +
  python projectors via the existing `load-file` modules; resolves

  per-cycle heartbeat hyperedge per root; dispatches each changed

- Wrote `futon3/holes/labs/M-live-geometric-stack/file-type-inventory.md`

prediction errors:

1. **`babashka.fs/watch` does not exist.** Initial multi-watcher
   used it; bb stdlib does not include a watcher. Available
   alternatives:
   - `pod-babashka-fswatcher` — bb pod, not installed
   - `inotifywait` (inotify-tools) — Linux native, not installed
   - polling via `walk-file-tree + mtime` — works today
   Chose polling (matches phase-4's working design). `inotify`
   upgrade can land as phase 4.6 once the dep is available
   (`sudo apt install inotify-tools` or pod install).

2. **Test-marker contamination from earlier phase-4 invariant test.**
   The smoke test surfaced a residual `watcher-test-marker-*.clj`
   file from the phase-4 test that wasn't fully cleaned. Phase-4's
   test does `fs/delete` but in the case of an interrupted run the
   file can persist. Phase-5's substrate-introspection signatures
   would catch this category as "concept introduced without
   structural attachment" — the file is in the substrate but the
   physical file is gone.

3. **Per-file ingest's `collect-repo` rebuild on every invocation
   is the throughput hot spot.** ~200ms per invocation on futon2;
   would be ~3-5s on futon3c-scale. Phase 4.6 caching opportunity:
   keep `by-ns` in process state across invocations within a
   watcher session, invalidate on any file change. Acceptable for
   v0; the watcher overhead per fs-event is bounded by this.

### What this delivers operationally

Joe's stated invariant — "save a pattern → it's in futon1a
within 1-2s" — is now achievable for any file type with a
projector. Concrete:

- Run: `bb futon3/scripts/multi_watcher.clj
        --root /home/joe/code/futon2=futon2-d
        --root /home/joe/code/futon3c=futon3c-d
        --root /home/joe/code/futon4=futon4-elisp-d
        --root /home/joe/code/futon1a=futon1a-d ...`
- Save any `.clj`/`.cljs`/`.cljc`/`.el`/`.py` (when Codex's
  Python projector lands) anywhere in the watched roots.
- Within 2s + ingest time (~200-500ms), the substrate reflects
  the change.
- Heartbeat hyperedges accumulate so any operator can see
  "watcher is alive and watching" without tail'ing logs.

### What still doesn't work (deferred to phase 4.6+)

- **`.flexiarg` patterns** — categorically the right thing to
  add next given Joe's framing; deferred to category-C refactor
  per `file-type-inventory.md`.
- **inotify-backed sub-second latency** — install
  `inotify-tools` and swap polling for `inotifywait` event
  stream.
- **`by-ns` cache** — keep resolution context in-process across
  per-file invocations.
- **Cascade staleness** (`:edge/witness-stale` flag on
  downstream edges) — needed before pattern-application
  consumers care.
- **Git post-commit trigger** — could complement polling for
  immediate commit-visibility.
